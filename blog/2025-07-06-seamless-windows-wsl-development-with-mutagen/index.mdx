# Seamless Windows-WSL development with Mutagen

The holy grail of Windows development: native filesystem performance on both Windows and WSL2, with real-time synchronization between them. This setup enables you to leverage Windows tools where they excel while maintaining blazing-fast filesystem I/O in WSL for Python, machine learning, and other Linux-native workloads.

<!-- truncate -->

## The problem with WSL filesystem performance

WSL2 has revolutionized Windows development, but it comes with a critical limitation: **cross-filesystem I/O is painfully slow**.

Reading Windows files from WSL (via `/mnt/c/`) results in terrible performance. While reading WSL files from Windows (via `\\wsl.localhost\`) is acceptable, it's still not native speed.

For developers working with large codebases, Python environments, or machine learning datasets, this performance gap is a deal-breaker.

## Why not just use VMs or dual boot?

I've tried both approaches, and here's why WSL2 with proper sync beats them:

### Virtual Machines (Hyper-V, VMware)
- ❌ **GPU passthrough complexity**: True GPU access requires dedicated hardware and complex setup
- ❌ **Resource overhead**: Running a full VM alongside Windows is resource-intensive
- ❌ **Context switching**: Constantly switching between host and guest OS interrupts workflow

### Dual Boot
- ❌ **Context switching overhead**: Rebooting between OSes kills productivity
- ❌ **Shared state issues**: Managing shared files and configurations between OSes is messy
- ❌ **Hardware compatibility**: Some hardware features may not work optimally in Linux

### WSL2 + Sync (This approach)
- ✅ **Native performance on both sides**: Files live natively where they're accessed most
- ✅ **Seamless GPU access**: WSL2 provides direct GPU passthrough for CUDA/ML workloads
- ✅ **Minimal overhead**: Real-time sync with negligible performance impact
- ✅ **Unified development environment**: No context switching, instant tool access

## Enter Mutagen: Real-time bidirectional sync

[Mutagen](https://mutagen.io/) is a real-time file synchronization tool designed for development workflows. Unlike traditional sync tools, it's built specifically for handling development files with features like:

- **Real-time synchronization** with filesystem watching
- **Conflict resolution** for simultaneous edits
- **Extensive ignore patterns** for node_modules, build artifacts, etc.
- **Cross-platform compatibility** including WSL2 integration

## Setting up the perfect sync

Let's walk through setting up a robust, production-ready sync between Windows and WSL2.

### Prerequisites

Before we start, ensure you have:

- Windows 10/11 with WSL2 enabled
- Ubuntu 22.04 (or your preferred WSL distro) installed
- Administrative access to Windows

<details>
  <summary>WSL2 installation reminder</summary>

  If you don't have WSL2 set up yet:

  ```powershell
  # Install WSL2 and Ubuntu
  wsl --install -d Ubuntu-22.04
  
  # Verify installation
  wsl --list --verbose
  ```

</details>

### Step 1: Install Mutagen

Download and install Mutagen on Windows:

<Image
  img={require('./assets/mutagen-download-page.png')}
  alt="Screenshot of Mutagen GitHub releases page highlighting the Windows download"
/>

```powershell
# Create directory and download (adjust version as needed)
mkdir C:\Users\$env:USERNAME\Bin
cd C:\Users\$env:USERNAME\Bin

# Download latest release
Invoke-WebRequest -Uri "https://github.com/mutagen-io/mutagen/releases/download/v0.17.5/mutagen_windows_amd64.zip" -OutFile "mutagen.zip"

# Extract
Expand-Archive -Path mutagen.zip -DestinationPath . -Force
Remove-Item mutagen.zip

# Add to PATH
[Environment]::SetEnvironmentVariable("Path", $env:Path + ";C:\Users\$env:USERNAME\Bin", [EnvironmentVariableTarget]::User)

# Verify installation
mutagen version
```

### Step 2: Configure ignore patterns

Create a comprehensive ignore configuration to avoid syncing unnecessary files:

```yaml
# ~/.mutagen.yml
sync:
  defaults:
    mode: "two-way-resolved"
    scanMode: "accelerated"
    watch:
      mode: "portable"
    ignore:
      vcs: true
      paths:
        # Node.js / JavaScript
        - "node_modules/"
        - ".next/"
        - "dist/"
        - "build/"
        - ".turbo/"
        - ".cache/"
        - "coverage/"
        
        # Python
        - "__pycache__/"
        - "venv/"
        - ".pytest_cache/"
        - ".mypy_cache/"
        
        # Build artifacts
        - "*.log"
        - "*.tmp"
        - ".env.local"
        
        # IDEs
        - ".idea/"
        - ".vscode/"
        
        # OS files
        - ".DS_Store"
        - "Thumbs.db"
    permissions:
      defaultFileMode: 0644
      defaultDirectoryMode: 0755
```

<Image
  img={require('./assets/mutagen-config-file.png')}
  alt="Text editor showing the .mutagen.yml configuration file with syntax highlighting"
/>

:::info

The `vcs: true` setting automatically ignores version control directories like `.git/`, which is crucial for avoiding sync conflicts with Git operations.

:::

### Step 3: Create the sync session

Now we'll create the actual sync between your Windows and WSL repositories:

```powershell
# Start the Mutagen daemon
mutagen daemon start

# Create the sync session
mutagen sync create `
  "C:\Users\$env:USERNAME\Repositories" `
  "\\wsl.localhost\Ubuntu-22.04\home\$env:USERNAME\Repositories" `
  --name="repositories" `
  --scan-mode="accelerated" `
  --watch-mode="portable"

# Verify sync creation
mutagen sync list
```

  img={require('./assets/sync-creation-success.png')}

### Step 4: Automate startup

To ensure sync resumes after reboots, we'll create startup automation:

<details>
  <summary>Create startup script</summary>

  Create `mutagen-startup.bat`:

  ```batch
  @echo off
  mutagen daemon start
  timeout /t 2 /nobreak >nul
  mutagen sync resume --all
  if not "%1"=="min" start /min cmd /c "%~0" min & exit
  ```

  Register for startup:

  ```powershell
  # Create startup shortcut
  $WshShell = New-Object -comObject WScript.Shell
  $Shortcut = $WshShell.CreateShortcut("$env:APPDATA\Microsoft\Windows\Start Menu\Programs\Startup\Mutagen.lnk")
  $Shortcut.TargetPath = "C:\Users\$env:USERNAME\Bin\mutagen-startup.bat"
  $Shortcut.WindowStyle = 7  # Minimized
  $Shortcut.Save()
  ```

</details>

## Troubleshooting common issues

### Sync appears stuck

If your sync seems to hang during the initial scan:

```powershell
# Check for scan problems
mutagen sync list -l repositories

# Force a flush if needed
mutagen sync flush repositories
```

<Image
  img={require('./assets/troubleshooting-scan-problems.png')}
  alt="Terminal output showing scan problems with file paths and suggested solutions"
/>

### High CPU usage during sync

This is normal during the initial sync but shouldn't persist:

```powershell
# Pause sync temporarily
mutagen sync pause repositories

# Resume when ready
mutagen sync resume repositories
```

### Permissions issues

If you encounter permission problems:

```bash
# In WSL, fix ownership
sudo chown -R $USER:$USER ~/Repositories
```
